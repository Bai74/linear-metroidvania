# 练习项目：《最后一盏灯》

一个 5~10 分钟的微型体验，覆盖主项目所需的全部核心技能。

---

## 项目定位

| 维度 | 设定 |
|------|------|
| 目标 | 学会 Godot 的核心工作流，产出一个可玩的小体验 |
| 时长 | 制作周期 1~2 周（每天 1~2 小时） |
| 游玩时长 | 5~10 分钟 |
| 角色 | 1 个（孩子） |
| 房间 | 5 间 |
| 谜题 | 2 个简单谜题 |
| 对话/文本 | 3~5 段环境文字 |
| 结局 | 到达终点即完成 |

---

## 故事设定

> 一个孩子在黑暗中醒来，面前只有一盏快要熄灭的油灯。他拿起灯，穿过五个房间，最终走到一扇门前。门上写着一行字。他推开门，光涌了进来。

**没有复杂剧情，没有多角色，没有分支**——就是一个"从黑暗走向光"的简单体验。

但这个体验需要用到主项目的每一个核心技能。

---

## 五个房间设计

```
房间1          房间2          戸间3          房间4          房间5
[醒来]   →    [拿灯]    →    [解谜1]   →    [解谜2]   →    [终点]
 学习移动      学习光照       学习交互        学习综合       学习叙事演出
```

### 房间 1：黑暗中醒来

| 内容 | 说明 |
|------|------|
| 视觉 | 几乎全黑，只有极微弱的环境光 |
| 玩家状态 | 只能左右移动和跳跃 |
| 叙事 | 地上有 1 个可交互物品（一张纸条），上面写着"拿着灯，往前走" |
| 学习目标 | 玩家移动、基础 Tilemap、交互提示 UI |

### 房间 2：找到灯

| 内容 | 说明 |
|------|------|
| 视觉 | 房间中央有一盏油灯在发光（PointLight2D + 闪烁） |
| 事件 | 靠近油灯 → 自动拾取 → 角色周围出现跟随光源 → 整个房间被照亮 |
| 揭示 | 灯亮后可以看到墙壁上的涂鸦（环境叙事元素） |
| 学习目标 | PointLight2D 跟随角色、拾取事件、灯光揭示隐藏内容 |

### 房间 3：第一个谜题（推箱子）

| 内容 | 说明 |
|------|------|
| 视觉 | 灯光范围有限，需要走近才能看清结构 |
| 谜题 | 一个石块挡住去路 → 推开石块 → 露出下方的地板开关 → 踩上去 → 门打开 |
| 威胁 | 无 |
| 学习目标 | 可推动物体、压力板机关、门的开关逻辑 |

### 房间 4：第二个谜题（找钥匙）

| 内容 | 说明 |
|------|------|
| 视觉 | 一个上锁的门 + 房间内散落的杂物 |
| 谜题 | 在杂物中找到钥匙（需要走近检查 2~3 个物品，钥匙在最后一个） → 用钥匙开门 |
| 叙事 | 散落的物品旁有简短文字（"一张全家福，但人脸被划掉了"） |
| 学习目标 | 物品交互系统、钥匙+锁逻辑、环境叙事文本显示 |

### 房间 5：终点

| 内容 | 说明 |
|------|------|
| 视觉 | 一扇巨大的门，门上有一行字 |
| 事件 | 走近门 → 文字显现："你已经走了很远了" → 推开门 → 画面从暗到亮渐变 → 黑屏 → 结束 |
| 学习目标 | 场景事件触发、画面渐变特效、游戏结束流程 |

## 技术学习路线（10 步）

每一步对应一个可验收的成果，完成后才能进入下一步。

### 第 1 步：项目搭建与角色移动

**学习内容**：Godot 基础概念（场景、节点、脚本、信号）

**具体任务**：
- [ ] 创建 Godot 4.x 项目
- [ ] 创建一个 CharacterBody2D 节点作为玩家
- [ ] 添加一个简单的矩形碰撞体（不用美术资源，用 ColorRect 就行）
- [ ] 编写移动脚本：左右移动 + 跳跃
- [ ] 加入 Coyote Time 和 Jump Buffer
- [ ] 创建一个简单的地面（StaticBody2D + 矩形）

**验收标准**：方块角色可以在地面上左右移动和跳跃，手感流畅

**预计时间**：2~3 小时

---

### 第 2 步：Tilemap 与房间搭建

**学习内容**：TileMapLayer 节点、TileSet 资源、房间结构

**具体任务**：
- [ ] 创建一个 16×16 的简单 TileSet（用纯色方块即可，不需要美术）
- [ ] 用 TileMapLayer 搭建房间 1 的地形（地面、墙壁、平台）
- [ ] 设置摄像机边界（不让摄像机看到房间外）
- [ ] 在房间 1 放置 1 个可交互物品（纸条）

**验收标准**：角色在搭建好的房间内移动，摄像机不越界

**预计时间**：2~3 小时

---

### 第 3 步：光照系统

**学习内容**：Light2D、CanvasModulate、光影氛围

**具体任务**：
- [ ] 添加 CanvasModulate 节点，将全局环境光设为接近全黑（如 Color(0.05, 0.05, 0.08)）
- [ ] 在房间 2 中放置一个固定光源（油灯），使用 PointLight2D
- [ ] 给油灯添加闪烁效果（用 Tween 或代码控制 energy 在 0.8~1.2 间波动）
- [ ] 实现角色跟随光源（拾取油灯后，PointLight2D 移动到角色节点下）
- [ ] 观察效果：灯光照亮的范围内可以看到更多细节，范围外几乎全黑

**验收标准**：拾取油灯后，角色周围出现跟随的暖色光源，光影效果有氛围感

**预计时间**：2~3 小时

---

### 第 4 步：交互系统

**学习内容**：Area2D、信号、UI 显示

**具体任务**：
- [ ] 创建一个可交互物品基类（Area2D + 碰撞体 + 交互逻辑）
- [ ] 实现靠近物品时显示交互提示（按 E 交互）
- [ ] 实现交互后触发事件（拾取物品 / 显示文本 / 开门）
- [ ] 在房间 1 放置纸条，交互后显示文字
- [ ] 在房间 2 放置油灯，交互后触发拾取事件

**验收标准**：靠近物品时出现提示，按键后触发正确事件

**预计时间**：2~3 小时

---

### 第 5 步：对话文本系统

**学习内容**：UI Label、打字机效果、文本控制

**具体任务**：
- [ ] 创建一个对话框 UI（屏幕底部，无框线，纯文本）
- [ ] 实现打字机效果（逐字显示，每字 0.03~0.05 秒）
- [ ] 实现按任意键跳过/继续
- [ ] 在房间 4 的物品旁放置环境叙事文本
- [ ] 在房间 5 的门上放置最终文字

**验收标准**：靠近叙事物品交互后，屏幕底部逐字显示文本，按任意键可跳过

**预计时间**：2~3 小时

---

### 第 6 步：解谜系统（推箱子 + 压力板）

**学习内容**：物理交互、状态管理、条件判断

**具体任务**：
- [ ] 创建可推动石块（RigidBody2D 或用代码模拟推动逻辑）
- [ ] 创建压力板（Area2D，检测石块是否在上面）
- [ ] 创建门（有开/关两种状态的 StaticBody2D）
- [ ] 连接逻辑：石块在压力板上 → 门打开
- [ ] 搭建房间 3 的谜题布局

**验收标准**：推开石块到压力板上，门自动打开，角色可以通过

**预计时间**：2~3 小时

---

### 第 7 步：解谜系统（钥匙 + 锁）

**学习内容**：物品持有逻辑、条件解锁

**具体任务**：
- [ ] 实现物品持有系统（拾取钥匙后，角色"持有"钥匙）
- [ ] 创建上锁的门（交互时检查是否持有钥匙）
- [ ] 搭建房间 4：上锁的门 + 散落的可检查物品（2~3 个假物品 + 1 个真钥匙）
- [ ] 检查假物品时显示叙事文本，检查真钥匙时拾取
- [ ] 用钥匙交互上锁的门 → 门打开

**验收标准**：在房间 4 中找到钥匙，用钥匙开门通过

**预计时间**：2~3 小时

---

### 第 8 步：房间切换

**学习内容**：场景管理、SceneTree、房间过渡

**具体任务**：
- [ ] 将每个房间做成独立场景（.tscn 文件）
- [ ] 实现房间之间的过渡（走到边缘 → 画面渐黑 → 加载下一房间 → 画面渐亮）
- [ ] 设置每个房间的入口位置（角色出现在正确的位置）
- [ ] 连接 5 个房间的完整流程

**验收标准**：从房间 1 走到房间 5，每个过渡自然流畅

**预计时间**：2~3 小时

---

### 第 9 步：音频

**学习内容**：AudioStreamPlayer、音效触发、环境音

**具体任务**：
- [ ] 给角色添加脚步声（移动时播放，静止时停止）
- [ ] 给交互物品添加交互音效
- [ ] 给房间 2 的油灯添加火焰音效（循环播放）
- [ ] 添加一个全局环境音（低频嗡鸣或风声，循环播放）
- [ ] 在房间 5 开门时播放一个特殊的音效（如上升音阶）

**验收标准**：全程有音频反馈，不同房间有不同的环境音氛围

**预计时间**：1~2 小时（音效可以从 freesound.org 下载）

---

### 第 10 步：打磨与画面效果

**学习内容**：Shader 基础、粒子系统、画面特效

**具体任务**：
- [ ] 添加全局暗角效果（可以用 TextureRect + 径向渐变纹理实现，不需要 shader）
- [ ] 在油灯旁添加少量灰尘粒子（GPUParticles2D，每秒 2~3 个）
- [ ] 实现房间 5 开门时的画面渐变（从暗到亮，Tween 控制 CanvasModulate）
- [ ] 实现游戏结束画面（黑屏 + "谢谢游玩" 文字，停留 3 秒后可退出）
- [ ] 全流程测试一遍，修复 bug

**验收标准**：从头到尾玩一遍，5~10 分钟，体验完整，没有明显 bug

**预计时间**：2~3 小时

| 画面特效 + 粒子 | 画面特效系统（第七章） |
| 游戏结束流程 | 结局演出（第二章） |

## 完成后的下一步

练习项目做完后，你有两个选择：

| 选择 | 说明 |
|:----:|:-----|
| **A：直接开始主项目 MVP** | 用练习项目中学到的技能，开始制作《无名之渊》的序章（孩子角色） |
| **B：做一个扩展练习** | 给练习项目加入：敌人巡逻 AI + 追逐系统 + 恐惧画面效果，再练一轮 |

建议选 A——因为练习项目和主项目的 MVP（序章+孩子角色）在技术上高度重叠，直接开始主项目可以避免重复劳动。
