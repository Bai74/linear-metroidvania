# 线性银河恶魔城游戏 — 设计需求总纲

> 引擎：Godot | 类型：2D 横版银河恶魔城 | 叙事模式：线性剧情驱动
> 进度机制：找到特定 NPC 触发下一步

---

## 一、叙事系统设计

### 1.1 剧情门锁（Narrative Gating）

| 层级 | 机制 | 说明 |
|------|------|------|
| **大章节分界** | 剧情锁 | 打完当前区域 Boss + 找到关键 NPC → 解锁下一章节区域 |
| **章节内探索** | 能力锁 | 获得新能力（冲刺、钩爪等）→ 解锁区域内的新路径 |
| **支线/lore** | 探索锁 | 发现隐藏房间、收集物品 → 补充世界观但不影响主线 |

**核心原则**：剧情锁控制大方向，能力锁提供探索乐趣，二者互不冲突。

### 1.2 NPC 触发系统

#### NPC 存在形式

| 形式 | 适用场景 | 开发成本 |
|------|---------|---------|
| **活人角色** | 关键剧情转折、情感冲击 | 高（需要对话树、动画、配音） |
| **幽灵/投影** | 已故角色的回忆、历史信息 | 中（无需完整建模，但需视觉表达） |
| **遗留物品/录音** | 补充 lore、环境叙事 | 低（纯文本/音频，开发最快） |

#### 推荐搭配

- **主线关键节点**：活人 NPC（玩家与角色产生情感连接）
- **区域推进线索**：遗留物品/录音（提供方向感但不打断节奏）
- **情感高潮**：幽灵/投影（制造悲剧氛围或揭示真相）

### 1.3 NPC 位置与引导设计

| 引导强度 | 方式 | 适用场景 |
|---------|------|---------|
| **弱引导** | 地域名/特征描述（"北边的废墟"） | 主线 NPC（保留探索感） |
| **中引导** | 其他 NPC 对话提及、文档线索 | 玩家卡住时的自然提示 |
| **强引导** | 地图标记、光效提示、感叹号 | 第一次找不到后的保底机制 |

**防卡关机制**：

1. 玩家在同一区域逗留超过 X 分钟 → 触发提示事件
2. 查看任务日志 → 显示更详细的位置描述
3. 与区域内的其他 NPC 对话 → 可获取重复线索
4. **最终保底**：穷尽所有探索后，NPC 位置自动高亮

### 1.4 剧情状态机

```
[序章] ──NPC_A(引导者)──→ [第一章：觉醒]
                                │
                            Boss_1 + NPC_B
                                │
                            [第二章：探索]
                                │
                        NPC_C + NPC_D（可任意顺序）
                                │
                            [第三章：真相]
                                │
                            Boss_2 + NPC_E（关键转折）
                                │
                            [第四章：抉择]
                                │
                            NPC_F（多分支对话）
                                │
                          ┌─────┴─────┐
                        [结局A]     [结局B]
```

**设计要点**：
- 每个状态需要定义：前置条件、触发事件、解锁内容
- 状态切换时要有明确的视觉/音频反馈
- 不可逆选择前必须有警告提示

## 二、世界与关卡设计

### 2.1 地图规模建议

| 指标 | 推荐范围 | 说明 |
|------|---------|------|
| 总房间数 | 30~60 间 | 线性游戏不宜过大，避免叙事稀释 |
| 单房间尺寸 | 1~3 屏（1920×1080 为基准） | 银河恶魔城标准 |
| 游玩时长 | 6~12 小时 | 主线 6~8h + 探索/收集 2~4h |
| 回溯次数 | 3~5 次 | 每获得一个新能力触发一次 |

### 2.2 区域结构（建议 5 个主题区域）

```
区域1：起始之地（教学区）
  ├── 教学：基础移动、跳跃、攻击
  ├── NPC_A：引导者，交代世界观
  └── 出口：需要从 NPC_A 获得方向线索

区域2：废弃设施（能力解锁区）
  ├── 获得能力：冲刺/二段跳
  ├── 新路径：用新能力回到区域1解锁隐藏区域
  ├── NPC_B：揭示部分真相
  └── Boss_1：守门 Boss

区域3：地下迷宫（探索密集区）
  ├── 获得能力：钩爪/下砸
  ├── NPC_C + NPC_D：并行可找
  ├── 环境叙事：大量遗留物品
  └── 复杂路线设计，保留回溯乐趣

区域4：禁忌之地（剧情高潮区）
  ├── NPC_E：关键转折（牺牲/背叛/真相揭露）
  ├── Boss_2：情感驱动的战斗
  └── 视觉风格突变，强化剧情冲击

区域5：终焉之塔（结局区）
  ├── NPC_F：最终对话，分支选择
  ├── Boss_3（可选）：最终 Boss
  └── 结局A / 结局B
```

### 2.3 房间类型清单

| 类型 | 占比 | 说明 |
|------|------|------|
| **战斗房** | 30~35% | 敌人遭遇、小 Boss |
| **平台跳跃房** | 25~30% | 纯移动挑战 |
| **探索/解谜房** | 15~20% | 谜题、机关、隐藏通路 |
| **叙事房** | 10~15% | NPC 所在、剧情演出、存档点 |
| **过渡房** | 5~10% | 区域连接、氛围过渡 |

### 2.4 能力通行逻辑表

| 能力 | 通行条件 | 解锁区域示例 |
|------|---------|-------------|
| 基础跳跃 | 无 | 所有基础平台 |
| 二段跳 | 区域2获得 | 高处平台、区域1隐藏区域 |
| 冲刺 | 区域2获得 | 穿过窄裂缝、区域3捷径 |
| 钩爪 | 区域3获得 | 抓挂点跨越间隙、区域2上层 |
| 下砸 | 区域3获得 | 砸碎脆弱地板、地下通道 |

## 三、角色与动作系统

### 3.1 主角基础动作参数表

| 参数 | 推荐值 | 说明 |
|------|-------|------|
| 移动速度 | 200~300 px/s | 平台跳跃标准 |
| 跳跃高度 | 2.5~3 格（以 Tile 为单位） | 能跳上一层平台 |
| 二段跳高度 | 与首次相同或略低 | 不建议太高 |
| 冲刺速度 | 移动速度 ×2.5~3 | 短距离爆发 |
| 冲刺持续时间 | 0.2~0.3s | 够穿过一个裂缝 |
| 冲刺冷却 | 0.5~1.0s | 防止无限冲刺 |
| 攻击前摇 | 3~5 帧 | 手感关键 |
| 攻击持续 | 8~12 帧 | |
| 受击无敌帧 | 60~90 帧（1~1.5s） | 防止连续受伤 |

**手感参数（需要在开发中反复调试）**：

| 参数 | 说明 |
|------|------|
| Coyote Time | 离开平台后仍可跳跃的宽容时间（0.1~0.15s） |
| Jump Buffer | 提前按跳跃的缓冲时间（0.1~0.15s） |
| 重力曲线 | 上升慢、下降快（更可控的跳跃手感） |
| 冲刺取消 | 允许空中冲刺取消后接跳跃（技巧向） |

### 3.2 敌人行为分类

| 类型 | 行为模式 | 数量占比 |
|------|---------|---------|
| **巡逻型** | 固定路线来回移动 | 40% |
| **守卫型** | 在固定位置警戒，玩家进入范围后追击 | 25% |
| **伏击型** | 隐藏在环境中，玩家靠近后突然出现 | 15% |
| **飞行型** | 在空中按固定/随机路线移动 | 10% |
| **精英型** | 混合行为 + 更高血量 | 10% |

### 3.3 Boss 设计模板

每个 Boss 需要定义：

```
Boss 名称：
- 血量：
- 阶段数：
- 阶段1（100%~50%）：
  - 行为模式列表（攻击1、攻击2、巡逻、休息）

  - 攻击判定框大小
- 阶段2（50%~0%）：
  - 新增行为/加速/变形
  - 视觉/音效变化
```

## 五、叙事呈现方式

### 5.1 表现形式选择

| 场景 | 推荐形式 | 理由 |
|------|---------|------|
| 日常 NPC 对话 | 站桩对话 + 文本框 | 高频出现，需要高效制作 |
| 关键剧情转折 | 实时演出（in-engine） | 情感冲击，需要动画配合 |
| 世界观补充 | 环境叙事（散落的物品/文字） | 符合探索感，低成本高覆盖 |
| 过场动画 | 极少使用，仅开场/结局 | 高成本，线性游戏控制在 2~3 段 |
| 回忆/闪回 | 幽灵/投影 + 画面滤镜 | 低成本营造氛围 |

### 5.2 对话系统需求

- 打字机逐字显示效果
- 角色立绘/头像显示
- 对话选择分支（影响剧情走向或获得不同信息）
- 对话跳过/快进功能
- 对话历史记录（玩家可回看）
- 关键词高亮提示（重要信息用颜色标记）

## 六、节奏设计

### 6.1 标准游戏循环

```
探索房间 → 遭遇敌人 → 战斗 → 发现秘密 → 找到 NPC → 剧情推进 → 获得能力/线索 → 回溯探索 → Boss 战 → 剧情高潮 → 下一区域
```

### 6.2 单区域节奏模板（约 30~45 分钟）

```
[0~5min]   进入新区域，氛围建立，轻度战斗
[5~15min]  探索 + 解谜 + 收集，密度逐渐升高
[15~25min] 获得新能力，开始回溯
[25~35min] 用新能力解锁新路径 + 精英敌人
[35~40min] 找到关键 NPC，剧情推进
[40~45min] Boss 战 → 区域完成 → 进入下一区域
```

### 6.3 剧情密度建议

| 区域 | 剧情占比 | 战斗/探索占比 | 说明 |
|------|---------|-------------|------|
| 序章 | 40% | 60% | 重叙事，建立动机 |
| 前期 | 20% | 80% | 重探索，让玩家熟悉玩法 |
| 中期 | 25% | 75% | 剧情逐步揭示，节奏交替 |
| 后期 | 35% | 65% | 剧情加速，战斗密度上升 |
| 结局 | 50% | 50% | 重叙事，情感收束 |
